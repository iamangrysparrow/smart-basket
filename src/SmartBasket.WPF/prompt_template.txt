Извлеки данные чека в JSON.

ПРАВИЛА ДЛЯ ЧЕКА:
- shop: название магазина (АШАН, Пятёрочка и т.п.)
- order_datetime: дата со временем YYYY-MM-DD:hh:mm (год {{YEAR}} если не указан)
- order_number: номер заказа (например H03255764114)
- total: итоговая сумма (число)
- items: массив товаров

ПРАВИЛО ОПРЕДЕЛЕНИЯ ТОВАРА
- Товар всегда существительное в множественном или единственном числе! Никогда другие части речи! Нет существительного -> Пропусти!
- Строки "Собрано", "Оплата", "Доставка", "Сервисный сбор" - НЕ ТОВАР! ПРОПУСТИ!

ПРАВИЛА ДЛЯ КАЖДОГО ТОВАРА:
- name: полное название товара из чека (Если там есть единицы измерения, оставь их. Если нет единиц измерения - тоже оставь так)
- quantity: количество купленного (после названия и перед quantity_unit. если единицы измерения ПЕРЕД quantity то это часть названия товара)
- quantity_unit: единица измерения количества в чеке шт/кг/л и т.п. - по умолчанию "шт" (после quantity перед price)
- price: цена за единицу (после quantity_unit)
- amount: итоговая цена за позицию (quantity × price) (МОЖЕТ ОТСУТСТВОВАТЬ ЕСЛИ quantity = 1)
- unit_quantity: количество в единице товара (число) (После price или amount. Может отсутствовать. Тогда равна 0)
- unit: единица измерения товара (г/кг/мл/л/шт) - (после unit_quantity. Может отсутствовать. Тогда равна quantity_unit)

ИГНОРИРУЙ строки: "Собрано", "Оплата", "Доставка", "Сервисный сбор", телефоны, адреса.

JSON:
{"shop":"","order_datetime":"","order_number":"","total":0,"items":[{"name":"","quantity":1,"quantity_unit":"шт","price":0,"amount":0,"unit":"","unit_quantity":0}]}

ПРАВИЛО ФОРМИРОВАНИЯ ОТВЕТА
- "amount": "" -> ПРОПУСТИ! Это не товар!
- "price": "" -> ПРОПУСТИ! Это не товар!

Текст чека:
{{RECEIPT_TEXT}}
